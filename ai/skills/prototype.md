# Skill: prototype

Use para explorar rapidamente UX, copy, scoring, alertas, menu Telegram ou regra de negócio antes de mexer no runtime real.

## Objetivo

Aprender rápido com um experimento isolado e descartável.

## Processo

1. Declare a pergunta que o protótipo deve responder.
2. Crie algo simples de rodar.
3. Não use credenciais reais.
4. Não altere banco de produção.
5. Não acople ao scheduler ou sender real.
6. Registre o que foi aprendido.
7. Recomende implementar, ajustar ou descartar.

## Exemplos úteis

- Copy de alerta Telegram.
- Formato de score/raridade.
- Menu guiado.
- Simulação de limites Free/Premium.
- Ranking visual de anúncios.

## Saída esperada

- Protótipo isolado.
- Como rodar.
- Aprendizado.
- Próxima decisão.
