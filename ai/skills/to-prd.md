# Skill: to-prd

Use para features maiores: Mercado Pago/webhook, leilões v2, onboarding Premium, UX de busca, growth, Instagram/social ou source nova.

## Objetivo

Transformar uma ideia em PRD claro antes de virar código.

## Estrutura do PRD

- Problema.
- Público afetado.
- Objetivo de produto.
- Não objetivos.
- Fluxo Telegram esperado.
- Regras de negócio.
- Impacto técnico.
- Dados, tabelas e configs envolvidos.
- Riscos.
- Métricas de sucesso.
- Critérios de aceite.
- Plano incremental de implementação.

## Regras AutoHunter

- Não tratar API web como jornada principal do usuário final sem requisito explícito.
- Incluir impacto em plano Free/Premium quando afetar limite ou monetização.
- Para leilões, incluir disclosure e gates obrigatórios.
- Para pagamentos, não documentar automação como pronta se webhook não estiver implementado.

## Saída esperada

Um PRD copiável, objetivo e pronto para virar issues.
