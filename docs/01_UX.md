## Próximos itens de UX

### UX-01 — Digest semanal v2 (maior impacto de retenção)

**Problema:** digest atual não comunica o que o sistema fez quando não houve alerta. Usuário silencioso por 7 dias provavelmente abandonou.

**O que entregar:**

```
📋 Resumo da semana — Garagem Alvo

Civic Si manual · SP
Monitorei 847 anúncios — nenhum bateu seus critérios.
2 apareceram acima do seu limite de preço.

Golf GTI MK7
Sem anúncio novo esta semana.

Continuo monitorando. Aviso quando aparecer algo.
```

**Localização:** `app/scheduler/weekly_wishlist_digest_job.py` + `app/services/weekly_wishlist_digest_service.py`

**O que mudar:**
- Incluir `source_runs` da semana por wishlist para contar volume monitorado
- Incluir contagem de anúncios encontrados mas bloqueados por filtro (via `wishlist_listing_activity`)
- Separar "não encontrei nada" de "encontrei mas não passou no filtro de preço"
- Usar nome público "Garagem Alvo", não "AutoHunter"

**Critério:** usuário sem alerta entende que o sistema trabalhou.

### UX-02 — Contexto de raridade no alerta

**Problema:** alerta não diz se aquele carro é comum ou raro. "Civic Si 2019" pode aparecer 3x por semana ou 1x por mês — faz diferença para a urgência.

**O que entregar:** linha extra no alerta quando há dado:

```
🔍 2ª vez que aparece em 30 dias — anúncio raro
```
ou
```
📊 Modelo comum — aparece ~4x por semana em SP
```

**Fonte:** `wishlist_listing_activity` já tem dados de frequência por wishlist. Calcular média de aparições nos últimos 30 dias.

**Localização:** `app/notifications/telegram_formatter.py`

**Critério:** badge aparece quando há pelo menos 14 dias de histórico para a wishlist.

---

### UX-03 — Alerta de leilão: disclosure mais visível

**Problema:** o texto `Lance não é preço final.` existe mas aparece no rodapé. Em mensagem longa, usuário não lê.

**O que mudar:** mover o disclosure para antes do link, não depois:

```
⚠️ Lance não é preço final. Verifique edital e taxas.
🔗 Ver leilão
```

**Localização:** `app/services/auction_notification_service.py` + formatter de leilão

**Critério:** disclosure sempre visível sem scroll na mensagem.

### UX-05 — Onboarding: copy pós-criação mais específica por resultado

**Problema atual:** "Busca criada. Monitoramento iniciado." é genérico.

**O que entregar — 3 variantes:**

Variante A (achou imediatamente):
```
✅ Civic Si monitorado.
Já encontrei 4 anúncios — enviando agora.
```

Variante B (carro raro, sem resultado):
```
✅ Civic Si monitorado.
Nenhum anúncio agora — esse modelo aparece pouco.
Aviso na hora que aparecer.
```

Variante C (resultado será enviado em breve via fila):
```
✅ Civic Si monitorado.
Primeira varredura em andamento. Resultados chegam em minutos.
```

**Localização:** `app/bot/handlers_core.py` após confirmação de criação de wishlist
